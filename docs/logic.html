smabo — アプリ・ロジック仕様

ESP32 ロボット・コンパニオンアプリ (Flutter)。本書は実装されている動作ロジックの要約です。
概要 通信 メッセージ モード 顔 音声 センサ 走行 アーム ESP32設定 オーバーレイ

1. 概要・設計スタンス

  • スマホをロボットの「顔」にしつつ、走行/アームのコントローラ・センサ送信・音声対話を担う。
  • ROS ランタイム/ライブラリは未使用。通信は素の WebSocket + JSON のみ。JSON の形を rosbridge v2.0 互換に整形し、間に rosbridge_suite を挟めば ROS2 と相互運用できる、という立て付け。
  • ROS 互換の知識は lib/core/wire/ros_compat.dart 一箇所に隔離。転送層 ws_client.dart (WsClient) は ROS 非依存。

4. モードとナビゲーション

  • 専用メニューなし。PageView の左右スワイプで 顔 → 走行 → アーム を切替。
  • ジョイスティック等のコントローラに触れている間は PageView のスワイプを無効化（pageLocked）し、操作がページめくりに取られないようにする。

5. 顔（目）

描画（EyePainter）

  • 白目（真円）＋虹彩（外）＋瞳孔（内）＋ハイライトの構成。各要素は色または画像。
  • 瞬き＝上から背景色（＝瞼）が降りて目を覆う。背景画像にも追従。
  • 整合性：瞳孔 ≤ 虹彩 ≤ 白目 を強制（FaceConfig.clamped()）。
  • 瞳の可動域を再計算：maxOff = 白目半径 − 虹彩半径 で、視線移動でも白目から飛び出さない（＋円でクリップ）。

視線モード

  • ランダム：一定間隔でランダムな注視点へなめらかに移動。
  • 追従：/look_at の Pose を画面注視方向へ射影して追従。

ウェイク反応

  • 「smabo」検出で 虹彩の色だけ変化（サイズ不変）。検出を 1 パルスとして受け、約 2 秒保持。
  • 音声受付中は虹彩の色相を連続サイクル。
  • 画面復帰時の誤発火防止のため、生成時に現在の wakeCount をシード。

カスタマイズ（Face settings／横向き）

  • 顔画面の「Face settings」から遷移。左：ライブプレビュー／右：設定。
  • プレビューは実画面と同じアスペクト比の仮想キャンバスに描き FittedBox で縮小（WYSIWYG）。
  • 色は右ペイン内インラインで Wheel / RGB / HSV 指定、変更は即プレビュー反映。
  • サイズはスライダー＋テキスト両方。各要素に画像読込（image_picker）可。背景色＝瞼色。

6. 音声

  • オンデバイス STT（speech_to_text）を単一の連続セッションで運用。
  • ウェイク照合：ja ロケールでは「smabo」がカタカナ化されるため、 smabosumaboスマボ すまぼスマホすまほ のいずれかを含めば検出。
  • コマンド抽出：最終テキスト中のウェイク語の直後以降を採用（固定位置で切らないので先頭が欠けない）＋変種除去。
  • Siri 風：listenFor 30s / pauseFor 4s で話している間受付。finalResult が来なくても status: done で確定（取りこぼし防止・受付が固まらない）。
  • 結果 → /speech/recognized。逆に /speech/say 受信で TTS 読み上げ。
  • ON/OFF トグルあり。OFF 時は待受せず「smabo と話しかけてください」も非表示。OFF 中はマイク権限要求もしない。
  • 手動フォールバック：顔をシングルタップで即音声受付。

7. センサ送信

センサ内容
IMU加速度・ジャイロ＋相補フィルタで姿勢推定 → quaternion。sensor_msgs/Imu をレート指定で送信。
GPS位置ストリーム → NavSatFix（共分散は水平精度から近似）。権限要求あり。
カメラCompressed(JPEG) / raw(mono8) 切替、FPS/品質設定。フロント/バック選択（既定フロント、切替時は再起動）。
  • 各センサはどのモードからでも ON/OFF。設定画面にライブプレビュー（カメラ映像／IMU値／GPS値、StreamBuilder）。
  • 縦画面ではカメラ比率を 1/aspectRatio に補正。

8. 走行コントローラ（cmd_vel）

  • ジョイスティック → 10Hz で geometry_msgs/Twist 送信（デッドマン 0.5s 内）。
  • 縦＝前後（linear.x）、横＝旋回（angular.z）。縦ニュートラルで横に倒すとその場回転。
  • 縦デッドゾーン（|y|<0.12 は前後 0）でほぼ水平時は純回転。
  • 後退時の左右反転：明確に後退（y<−0.2）のときだけ旋回符号を反転 → 車のバックのように自然。0 境界では反転しないので振り切り時に逆回転しない。
final fwd = _y.abs() < 0.12 ? 0.0 : _y;
final linear  = fwd * maxLinear;
var   angular = -_x * maxAngular;
if (fwd < -0.2) angular = -angular;   // 自然な後退ステア
sendCmdVel(linear, angular);

9. アームコントローラ

  • ESP32 設定の servos.joints から manual サーボ（ランダムグループ非所属）だけを動的にスライダ表示。
  • 各 joint の min_angle/max_angle を使用。度→ラジアン変換し /servo/command（JointTrajectory）。
  • 並び順はアプリ側の設定順に追従。ホームは各 joint の init_angle。

10. ESP32 設定（rosbridge）

  • get_config で取得、set_config/set_mode で更新。ESP32 は更新後の設定を返さないため 送信時にローカルへ deep-mergeして即時反映（表示が戻らない）。
  • サーボ増減：追加＝新 joint にスペック付与。削除＝joint を null 送信（削除エンコード）。
  • 並べ替え：▲▼（アプリ側の表示順。ファームの dict 順は不変のため）。
  • manual / random：所属グループで決まる。
    • グループ所属＝random（自動・同グループは同タイミング発火、角度は各自独立）。
    • 非所属＝manual（手動コマンドのみ）。
    • 「Enable random motion」がランダム動作エンジンのマスタースイッチ。manual と random は共存。
  • UI：各サーボに「Manual / Random:<group>」割当、グループの追加/削除/interval 編集。 ピン/バス/WiFi は再起動を伴うため Advanced に隔離。

11. オーバーレイ自動非表示・設定画面

  • 上部ステータス（接続ドット・音声）／下部センサ・設定ボタン／ページドットは 全画面で一定時間後に自動非表示。ダブルタップで再表示（単発操作で勝手に出ない）。
  • 設定画面は縦向き（他は横向き）。最上部に目次（クリックでジャンプ）。
  • 言語は選択式（TTS/STT ロケール）。UI は英語表記。
